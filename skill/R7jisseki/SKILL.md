# R7実績データ取込スキル

## リファレンス

- **`references/reference_output.json`** — データスキーマ（キー名と型）。このファイルのキー名・構造に合わせてJSONを書く。

## 基本方針

**統計表と加算表のPDFに記載されている全ての項目・値をJSONに保存する。**
**スキルに従って行動する。自分の考えで勝手に行動しない。**

## フロー

```
PDF → Claudeが全項目を読み取り → data/r7_extracted.json に書き出し → ブラウザでR7実績ページで「実績読込」ボタンを押して読み込み → バリデーション → ユーザー確認
```

**JSONファイル名は `data/r7_extracted.json` 固定。** 毎回同じファイルを上書きする。

## 手順

### 1. PDFを読む

PDFディレクトリ（`PDF/`）にある統計表・加算表のPDFを読む。

### 2. Claudeが全項目を読み取る

PDFに記載されている**全ての行・全ての数値**を読み取る。

#### 読み取りルール

- **PDFに書いてある全ての項目を読み取る。スキーマにないものも含む**
- **スキーマにないものもがあればユーザーに報告しdata/r7_extracted.jsonに追加するか指示を仰ぐ**
- **スキーマにないものもがあればユーザーに報告し、次回のためにreference_output.jsonに追加するか指示を仰ぐ**
- **0は有効な値**。PDFに「0件」「0円」と書いてあれば0を保存する。
- **PDFに記載がない項目はJSONに入れない（nullのまま）**
- **数値のカンマは除去。円は整数、%は小数で保存**
- **絶対にPDFにない値を勝手に作るな。**
- **数値型を厳守**。文字列で保存しない。
- **加算種別内訳表の薬剤調製料加算の項目は合計だけでよい**

### 3. JSONファイルに書き出し

読み取ったデータを **`data/r7_extracted.json`** に書き出す。

`references/reference_output.json` のスキーマに合わせてr6オブジェクトを構築する。reference_output.jsonの各キーに対応する値をPDFから読み取った値で埋める。

### 4. ブラウザ（R7実績ページ）へ読み込み

1. Claude in Chromeを使いブラウザでindex.htmlを開く（プレビューサーバー経由）
2. 「報酬改定」タブ →「R6 R7実績」サブタブを開く
3. 「実績読込」ボタンを押してJSONを読み込む
4. 各セクションの値がブラウザに表示されることを確認する
5. PDF（加算種別内訳表）を見てS、"算定なし"となっているプルダウンで適切な項目を選択する

**ユーザーがHTMLを開いた時に数字や選択が入っている状態にすること**

### 5. バリデーション

ブラウザ（R7実績ページ）に表示された値をPDF記載値と照合する。

| ① | A. 調剤基本料・体制加算合計 = t_kihon_totalを確認 |
| ② | B. 薬剤調製料　薬剤料合計 = t_yakuzai_totalを確認 |
| ③ | B. 薬剤調製料　調製料合計 = t_chozai_total = k_chozai_grand_totalを確認 |
| ④ | B. 薬剤調製料　薬剤調製料加算合計 = t_kazan_total = k_kazan_grand_totalを確認 |
| ⑤ | C. 薬学管理料合計 = t_yakugaku_total = k_yakugaku_grand_total+k_zaitaku_grand_totalを確認 |
| ⑥ | D. 在宅等合計 = t_yakuzai_totalを確認 |
| ⑦ | E. 介護（単位制）合計 = k_kaigo_grand_totalを確認 |
| ⑧ | F. 患者負担合計 = t_hoken_futan+t_jihi_futan+t_hokengai_futanを確認 |
| ⑨ | 年間合計　医療保険（A+B+C+D）= t_total_rewardを確認 |
| ⑩ | null項目チェック | `data/r7_extracted.json` 内の全nullキーを列挙し、PDFで0件、0円か記載なしか確認。0件、0円なら0に修正。 |

## ユーザーへ報告

- エラーやうまく行かない問題は全てユーザーへ報告し、判断を仰ぐ。勝手に先へ進めない
- バリデーションの結果を報告
