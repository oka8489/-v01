# タスク管理スキル

## データファイル

**`data/tasks.json`** — 唯一のデータソース。このファイルを読み書きする。

### 構造

```json
{
  "categories": [
    { "id": "todoke", "label": "届出の変更・新規・取り下げ", "keys": ["t01", "u1234"] }
  ],
  "tasks": {
    "t01": {
      "title": "調剤基本料の届出更新",
      "detail": "受付回数・集中率・グループ規模を再計算し、届出を再提出。",
      "deadline": "R8.6.1（施行日）",
      "status": "todo"
    }
  }
}
```

### フィールド

- `title` — タスク名（必須）
- `detail` — 詳細説明（任意）
- `deadline` — 期限。令和表記 `R8.6.1` または自由テキスト（任意）
- `status` — `todo`（未着手）/ `wip`（進行中）/ `done`（完了）

### ID規則

- 定義済みタスク: `t01`, `t02`, ...
- ユーザー追加タスク: `u` + タイムスタンプ（例: `u1775054800723`）

## 操作

### 1. 一覧表示

data/tasks.jsonを読んで、カテゴリごとにタスクを表示する。ステータスも表示。

```
ユーザー: 「タスク一覧」「今のタスク見せて」
```

### 2. タスク追加

```
ユーザー: 「レセコン更新のタスクを追加して」「届出関連のタスクを全部追加して」
```

- IDは `u` + Date.now() で生成
- カテゴリが指定されなければ既存の最初のカテゴリに追加
- 新しいカテゴリが必要なら作成
- 一括追加も対応（ユーザーが複数タスクを指示した場合）

### 3. ステータス変更

```
ユーザー: 「調剤基本料を進行中にして」「レセコン更新を完了にして」
```

- タイトルの部分一致で検索
- `status`を `todo` / `wip` / `done` に変更

### 4. 編集

```
ユーザー: 「レセコン更新の期限を5月末に変更して」「調剤基本料の詳細を更新して」
```

- タイトルの部分一致で対象タスクを特定
- 指定フィールドを更新

### 5. 削除

```
ユーザー: 「レセコン更新のタスクを削除して」「完了タスクを全部削除して」
```

- 削除前に確認を取る
- `tasks`オブジェクトからキーを削除
- `categories`の`keys`配列からも削除

### 6. PDFからタスク生成（jissekiスキルと連携）

```
ユーザー: 「PDFを見て必要なタスクを追加して」
```

- 報酬改定関連のPDFを読み取り、必要な事務タスクを自動生成
- 既存タスクと重複しないよう確認

## 注意事項

- **data/tasks.jsonを直接編集する。** APIサーバー経由ではなくファイルを直接読み書きする。
- ブラウザはページリロードで最新のタスクを表示する（APIサーバーがdata/tasks.jsonを読むため）。
- 変更後、ユーザーに「ブラウザをリロードしてください」と伝える。
- JSONの文字コードはUTF-8、インデント2スペース。
